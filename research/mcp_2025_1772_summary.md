# IACR ePrint 2025/1772 论文总结：Multiple Concurrent Proposers: Why and How

## 1. 核心提出的问题
传统单提案者（Single-Proposer）区块链协议存在一个根本性的设计缺陷：**对交易包含权的串行垄断（Serial Monopoly）**。

这种暂时的绝对权力使得单个节点能够审查、重排或插入交易，从而提取巨大的经济租金，即**矿工可提取价值（MEV）**。文章将最赚钱的 MEV（如跨所套利、原子套利、清算）视作高频的链上拍卖。由于单提案者享有特权，当前在单个区块（时隙）内运行高效、公平的拍卖是几乎不可能的，原因在于提案者可以：
1. **审查（Censor）** 其他人的出价。
2. **偷看（Peek / No hiding）** 其他人的出价并据此更改自己的出价。
3. 拥有网络延迟带来的信息优势（Latency advantage）。

这些特权会导致严重的市场扭曲和逆向选择，极大地限制了链上金融应用的发展。

## 2. 文章的主要内容
为了解决上述市场失灵问题，本文提出了一种名为 **多并发提案者（Multiple Concurrent Proposers, 简称 MCP）** 的轻量级共识附加协议（gadget）。

该协议旨在为底层的区块链共识提供两个关键的经济学属性：
- **选择性审查抵抗（Selective-Censorship Resistance）**：确保提交给诚实节点的交易能够被立即包含，攻击者无法“选择性”地拖延或丢弃特定交易（即，要么包含所有交易，要么该轮次一个都不包含）。
- **隐藏性（Hiding）**：在共识最终确定之前，任何人都无法看到交易的具体内容，从而防止因“偷看”导致的逆向选择和抢跑。

文章的**核心贡献**在于：首次以独立于具体共识协议的方式，严格定义了上述两个属性，并设计出了一个可以挂载在任何标准 PBFT 风格共识协议之上的 MCP 协议。经过证明，该协议是安全且活跃的，在满足审查抵抗和隐藏性的同时，相比最优延迟协议，仅增加了一次网络往返的延迟开销。

## 3. 文章的逻辑梳理
1. **背景与动机**：指出 MEV 的根源在于单提案者垄断。通过将 MEV 提取抽象为链上拍卖问题，说明现有的修复手段治标不治本。
2. **博弈论案例分析（Three Examples）**：通过 Alice 和 Bob 参与拍卖的三个简化博弈场景，逐层剥析垄断特权对市场的破坏：
   - **场景 1（无审查抵抗，无隐藏）**：Bob 可以审查 Alice，导致 Bob 以零成本必胜，拍卖毫无意义。
   - **场景 2（有审查抵抗，无隐藏）**：Bob 无法审查，但能在确认前看到 Alice 的出价。Bob 可以进行完美的逆向选择（价格有利时加价赢走，不利时让给 Alice），导致 Alice 的最优策略是不出价，拍卖再次崩溃。
   - **场景 3（有审查抵抗，有隐藏）**：Bob 既不能审查也不能偷看，只能利用极小的延迟优势（最后看一眼信息）。此时双方必须真实博弈，拍卖恢复功能且效率大大提高。
3. **相关工作对比（Related Work）**：评估了此前的 MEV 缓解方案（如公平排序协议、加密内存池、交易批量处理、DAG 并发协议），指出它们在面对高级攻击时，都无法实现严格的“选择性审查抵抗”和“隐藏性”。
4. **密码学原语与模型定义（Preliminaries）**：引入构建协议所需的密码学工具，如具有隐藏属性的默克尔树（用于向量承诺）和 **隐藏纠删码（HECC）**，并形式化定义了部分同步网络模型和所需达成的经济学共识属性。
5. **协议设计与理论证明（Protocol & Proofs）**：详细提出 MCP 协议架构，并从数学和密码学上严格证明其满足安全性（Safety）、活跃性（Liveness）、选择性审查抵抗以及隐藏性。

## 4. 关键术语讲解
- **MEV (Miner Extractable Value / 矿工可提取价值)**：验证者/矿工利用其对区块内交易包含和排序的独占控制权，通过提前交易（front-running）、三明治攻击等方式从用户手中无风险套取的利润。
- **Serial Monopoly (串行垄断)**：在现有的区块链机制中，每个区块（时隙）由一个指定的领导者（单提案者）全权决定交易列表的市场结构。
- **Multiple Concurrent Proposers (MCP / 多并发提案者)**：一种允许多个节点在同一轮次中同时提出交易的协议设计，旨在打破单一提案者的审查垄断。
- **Selective-Censorship Resistance (选择性审查抵抗)**：一种严格的抗审查属性。它要求在当前时隙内，要么所有被诚实节点接收到的交易都被打包进区块，要么该区块不包含任何交易。攻击者不能精准挑出并丢弃对其不利的特定交易。
- **Hiding (隐藏性)**：在交易的最终顺序和包含状态被共识协议确定（达到 Valency 状态）之前，任何节点（包括提议者和恶意节点）都无法得知交易的具体明文内容。
- **HECC (Hiding Erasure-Correcting Codes / 隐藏纠删码)**：文章中使用的一种结合了秘密共享和 Reed-Solomon 纠删码的密码学原语。它将交易信息编码为多个分片，少于阈值数量的碎片无法泄露任何信息（提供隐藏性），而收集到足够数量的碎片则可以完全恢复原文。
- **Adverse Selection (逆向选择)**：在拍卖中，如果一方能偷看对方出价或拥有信息优势，劣势方就会意识到自己只有在做亏本买卖时才会“赢”，从而放弃参与，最终导致市场流动性枯竭。
